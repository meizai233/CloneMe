# CloneMe SaaS 平台测试用例

## 一、认证模块

### TC-AUTH-001 注册 - 正常流程
- 前置：未注册的邮箱
- 操作：填写邮箱、密码（≥6位）、名字，点击注册
- 预期：注册成功，自动登录跳转到工作台，左上角显示用户名

### TC-AUTH-002 注册 - 邮箱已存在
- 前置：已注册的邮箱
- 操作：用相同邮箱再次注册
- 预期：提示"该邮箱已注册"

### TC-AUTH-003 注册 - 参数校验
- 操作：邮箱为空 / 密码少于6位 / 名字为空
- 预期：前端校验不通过，无法提交

### TC-AUTH-004 登录 - 正常流程
- 前置：已注册账号
- 操作：输入正确的邮箱和密码，点击登录
- 预期：登录成功，跳转到工作台

### TC-AUTH-005 登录 - 密码错误
- 操作：输入正确邮箱 + 错误密码
- 预期：提示"邮箱或密码错误"

### TC-AUTH-006 登录 - 邮箱不存在
- 操作：输入未注册的邮箱
- 预期：提示"邮箱或密码错误"

### TC-AUTH-007 退出登录
- 前置：已登录状态
- 操作：点击右上角"退出"
- 预期：跳转到登录页，再访问 /dashboard 会重定向到登录页

### TC-AUTH-008 Token 过期
- 前置：手动修改 localStorage 中的 token 为无效值
- 操作：刷新页面
- 预期：跳转到登录页

---

## 二、数字人管理

### TC-AVATAR-001 创建数字人 - 正常流程
- 前置：已登录
- 操作：点击"创建数字人"，填写名称、描述、人设提示词，选择模型，点击保存
- 预期：创建成功，跳转到工作台，列表中出现新数字人

### TC-AVATAR-002 创建数字人 - 名称为空
- 操作：不填名称直接保存
- 预期：提示"名称不能为空"

### TC-AVATAR-003 创建数字人 - 达到数量上限
- 前置：已创建 3 个数字人（free 套餐上限）
- 操作：再创建一个
- 预期：提示"已达数字人数量上限（3个）"

### TC-AVATAR-004 编辑数字人
- 前置：已有数字人
- 操作：点击"编辑"，修改名称和人设，保存
- 预期：保存成功，工作台列表显示更新后的名称

### TC-AVATAR-005 删除数字人
- 前置：已有数字人
- 操作：点击"删除"，确认弹窗点击确定
- 预期：数字人从列表中消失

### TC-AVATAR-006 删除数字人 - 取消
- 操作：点击"删除"，确认弹窗点击取消
- 预期：数字人仍在列表中

### TC-AVATAR-007 数字人详情
- 操作：点击"编辑"进入详情页
- 预期：显示数字人的所有配置信息，包括声音状态、知识库数量

---

## 三、模型管理

### TC-MODEL-001 浏览模型商店
- 前置：已登录普通用户
- 操作：访问模型商店页面
- 预期：显示所有上架模型，免费模型标记"免费"，付费模型显示价格

### TC-MODEL-002 查看可用模型
- 前置：已登录，有部分模型已授权
- 操作：创建数字人时查看模型选择器
- 预期：只显示免费模型 + 已授权模型

### TC-MODEL-003 使用未授权模型创建数字人
- 前置：选择一个未授权的付费模型
- 操作：保存数字人
- 预期：提示"未授权使用此模型，请前往模型商店"

### TC-MODEL-004 管理员上传模型
- 前置：管理员账号登录
- 操作：填写模型名称、分类、价格、model_url，提交
- 预期：模型创建成功，出现在模型列表中

### TC-MODEL-005 管理员上下架模型
- 前置：管理员，已有上架模型
- 操作：点击"下架"
- 预期：模型状态变为 disabled，普通用户不可见

### TC-MODEL-006 管理员授权模型给租户
- 前置：管理员，已有付费模型和目标租户
- 操作：选择租户，设置有效期，点击授权
- 预期：授权成功，目标租户可以使用该模型

### TC-MODEL-007 管理员回收授权
- 前置：已有授权记录
- 操作：点击"回收"
- 预期：授权被删除，租户不再能使用该模型

---

## 四、声音克隆

### TC-VOICE-001 录音上传
- 前置：进入声音工坊
- 操作：点击"开始录音"，朗读参考文本 15 秒，点击"停止录音"
- 预期：显示"✅ 录音已上传"，音频 URL 自动填入

### TC-VOICE-002 创建克隆声音 - 正常
- 前置：已上传录音
- 操作：勾选授权确认，点击"创建克隆音色"
- 预期：显示"音色已就绪"+ voiceId

### TC-VOICE-003 创建克隆声音 - 录音太短
- 前置：录音不足 5 秒
- 操作：点击创建
- 预期：提示"录音时长太短，请录制 10~20 秒的语音样本后重试"

### TC-VOICE-004 创建克隆声音 - 未勾选授权
- 操作：不勾选授权确认直接创建
- 预期：提示"请先确认已获本人授权"

### TC-VOICE-005 清除音色
- 前置：已有克隆声音
- 操作：点击"清除音色"
- 预期：voiceId 清空，后续对话使用默认声音

---

## 五、对话功能

### TC-CHAT-001 文字对话 - 正常流程
- 前置：进入数字人对话页面
- 操作：输入问题，点击提问
- 预期：对话框显示"🤔 正在思考中..."，然后逐字显示回复文字

### TC-CHAT-002 流式输出
- 操作：提问后观察对话框
- 预期：文字逐字出现（非一次性显示），有打字机效果

### TC-CHAT-003 提问后输入框清空
- 操作：输入问题并提交
- 预期：输入框立即清空

### TC-CHAT-004 空问题校验
- 操作：不输入任何内容直接提问
- 预期：提示"请输入问题后再提问"

### TC-CHAT-005 多轮对话
- 操作：连续提问 3 个问题
- 预期：每次都能正常回复，对话框显示最新回复

---

## 六、TTS 语音合成

### TC-TTS-001 语音播放 - 有克隆声音
- 前置：已创建克隆声音
- 操作：提问并等待回复
- 预期：回复文字显示的同时，听到克隆声音朗读回复内容

### TC-TTS-002 语音播放 - 无克隆声音
- 前置：未创建克隆声音
- 操作：提问
- 预期：只有文字回复，无语音播放，Live2D 使用 fallback 口型动画

### TC-TTS-003 停止语音播放
- 前置：语音正在播放
- 操作：点击对话框右上角 ⏹ 按钮
- 预期：语音立即停止，口型动画停止

### TC-TTS-004 多句语音连续播放
- 前置：LLM 回复包含多句话
- 操作：等待语音播放
- 预期：多句话连续播放，句间无明显长停顿

### TC-TTS-005 多次对话后语音仍正常
- 操作：连续对话 5 次以上
- 预期：每次对话都能正常生成语音（TTS WebSocket 自动重连）

---

## 七、Live2D 数字人形象

### TC-LIVE2D-001 模型加载
- 操作：进入对话页面
- 预期：Live2D 模型正常显示，有待机动画（呼吸、眨眼）

### TC-LIVE2D-002 口型同步
- 前置：语音正在播放
- 操作：观察 Live2D 模型
- 预期：嘴巴随语音节奏开合

### TC-LIVE2D-003 表情切换
- 操作：提问一个让数字人开心的问题
- 预期：回复时 Live2D 模型表情变化（如微笑）

### TC-LIVE2D-004 模型加载失败降级
- 前置：删除 Live2D 模型文件
- 操作：进入对话页面
- 预期：显示 Mock Fallback 模式（简笔画脸），功能不受影响

## 八、多租户隔离

### TC-TENANT-001 数据隔离
- 前置：两个不同租户的账号 A 和 B
- 操作：A 创建数字人，B 登录查看
- 预期：B 看不到 A 的数字人

### TC-TENANT-002 模型授权隔离
- 前置：管理员只给租户 A 授权了某付费模型
- 操作：租户 B 尝试使用该模型创建数字人
- 预期：提示"未授权使用此模型"

### TC-TENANT-003 API 越权访问
- 操作：用租户 A 的 token 访问 `/api/avatars/{B的数字人ID}`
- 预期：返回 404（因为 tenant_id 不匹配）

---

## 九、异常场景

### TC-ERR-001 后端服务不可用
- 前置：停止后端服务
- 操作：前端尝试登录/对话
- 预期：显示"网络异常，请检查服务是否启动"

### TC-ERR-002 LLM 服务超时
- 前置：LLM 响应超过 5 分钟
- 操作：等待
- 预期：前端不崩溃，超时后显示错误提示

### TC-ERR-003 TTS WebSocket 断开
- 前置：对话过程中断开网络
- 操作：恢复网络后继续对话
- 预期：TTS 自动重连，后续对话语音正常

### TC-ERR-004 并发对话
- 操作：快速连续点击提问按钮
- 预期：不会出现重复回复或页面崩溃
